## 🎯 즉시 결재 필요 안건 (4건)

### 안건 1: ELS Hard Cap 추가 하향 검토

**현재**: MAX_HOTEL_BOOST = 1.25× (실질 마진 6.06% 기준)
**문제**: 실제 마진 4% 기준 시 Diamond + 1.25× boost = **마진 환원율 23.5%**
**옵션**:
- (a) 1.0× 로 cap (boost 폐기) → 마진 사수, 마케팅 효과 감소
- (b) 1.1× 로 cap (보수적 유지) → 환원율 16.1%
- (c) 1.25× 유지 + boost는 호텔 자부담 100% 필수 → DOTBIZ 영향 0
- **권장 (c)**: 호텔 자부담 비율 필드 신설, 100% 미만이면 등록 거부

### 안건 2: PG 전략 — 국가별 분리

**현재**: 단일 글로벌 PG (Eximbay 등) 전제 → 3.5~4% 수수료
**제안**: **국가별 Local PG 우선 + Global PG는 fallback**

| 국가 | Local PG 후보 | 추정 수수료 |
|------|--------------|------------|
| 한국 | Toss Payments / KCP / NICE / 이니시스 | **1.5~2.5%** |
| 일본 | GMO Payment / Stripe Japan | 2.5~3.4% |
| 중국 | Alipay / WeChat Pay (B2B 직결제) | 1.0~2.0% |
| 베트남 | VNPay / OnePay | 2.0~3.0% |
| 싱가포르 | Stripe / Adyen | 2.4~2.9% |
| 글로벌 fallback | Eximbay / Stripe | 3.5~4.0% |

**액션 아이템**:
- 📋 마켓 리서치 PRD 작성 (각 국가별 PG 후보 3종 비교)
- 📋 통합 결제 라우터 설계 (국가 → PG 자동 라우팅)
- 📋 각 PG 사 계약 협상 (CFO + 기획)

**예상 효과**: 평균 PG 수수료 3.5% → **2.0% 절감** (한국·중국 비중 높을수록)

### 안건 3: 카드 결제 정책 재설계

**현재**: Non-refundable은 무조건 PG 카드, 송금 불가
**문제**: PG 수수료 흡수 시 손실 → **가격 차등 필요**

**Sage 권장안 (2단계 보완)**:
| 케이스 | 결제 방식 | 가격 |
|--------|----------|------|
| Free Cancel + TL 미도래 | 송금 우선 / 카드 옵션 | **단일 표시가** |
| Non-refundable | PG 카드 강제 | **PG 수수료 반영 인상가** |
| TL 경과 | PG 카드 강제 | 동일 |

**핵심**: Non-refundable 가격은 **표시가 + 카드 결제 수수료 흡수분**을 사전에 포함.
- 호텔 마진 8~12% 활용해 PG 3% 흡수 후도 마진 양수 유지
- Free Cancel은 송금 가능하므로 PG 부담 없음

### 안건 4: 보안 강화 — 별도 트랙

**현재 상태**: 보안 취약 (대표이사 의견)

## 🔍 경쟁사 PG 모델 벤치마크 (Market Research)

### 대표이사 시장 조사 인풋

많은 B2B 공급사들이 소규모 업체용 시스템(DOTBIZ 유사)을 통해 고객사 결제를 받고 있음. 그중 두 거대 공급사가 **서로 다른 모델**을 사용:

| 공급사 | PG 모델 | 결제수수료 부담 | 가격 표시 |
|--------|---------|---------------|----------|
| **TBO** | 외부 청구 모델 | **고객사가 직접 부담** (별도 청구) | 호텔 정가 + 결제수수료 별도 |
| **Ratehawk** | 가격 포함 모델 | **호텔 가격에 포함** | 결제수수료 포함 단일 가격 |
| **DOTBIZ 현재 결정** | DOTBIZ 흡수 + 마진 반영 | DOTBIZ 흡수 | 가격에 반영 (Ratehawk 유사) |

### 두 모델 비교

#### TBO 방식 (고객사 부담)

| 영역 | 효과 |
|------|------|
| **DOTBIZ 마진** | ✅ 안전 (PG 0% 흡수) |
| **고객 인식** | ⚠️ 추가 비용 명시 (불만 가능) |
| **가격 경쟁력** | ✅ 호텔 가격 자체는 저렴 |
| **운영 복잡성** | ⚠️ 청구 항목 분리 (Invoice 라인 추가) |
| **결제수단 분기** | ✅ 송금/카드 별도 가격 가능 |

#### Ratehawk 방식 (가격 포함)

| 영역 | 효과 |
|------|------|
| **DOTBIZ 마진** | ⚠️ 호텔 마진/마크업 잠식 |
| **고객 인식** | ✅ 단일 가격 (UX 깔끔) |
| **가격 경쟁력** | ⚠️ 표시가 인상 (비교 시 불리) |
| **운영 복잡성** | ✅ 단순 |
| **결제수단 분기** | ❌ 어려움 (단일 가격) |

### 권장 — 하이브리드 모델 (시장 검증된 두 패턴 결합)

DOTBIZ가 두 방식 중 하나만 채택하기보다 **거래 유형/고객 규모별로 분기 운영** 권장:

```
                  ┌─ 단일 표시가 (호텔 가격 + PG 흡수)
                  │   (Ratehawk 방식)
   Free Cancel ──┤   Non-refundable (호텔 마진 8~12% 활용)
                  │
                  └─ 송금 옵션 (PG 0%) — 별도 안내

   POSTPAY 정산 ──── 송금 (PG 0%) — Net-30 청구 (TBO 방식 변형)

   대형 고객사 ──── 계약 협의로 결제수수료 별도 명시 (TBO 방식)
   (월 TTV 5억+)
```

### 모델별 적용 정책 (결재 결과 반영)

| 거래 유형 | 결제 방식 | PG 처리 | 모델 |
|----------|----------|---------|------|
| **Free Cancel + TL 미도래** | 송금 우선 | 0% (DOTBIZ/고객 부담 없음) | — |
| 동 + 카드 결제 선택 | 카드 | 단일 표시가에 흡수 | Ratehawk |
| **Non-refundable** | 카드 강제 | 호텔 마진 활용 흡수 | Ratehawk |
| **POSTPAY 정산** | 송금 (Net-30) | 0% | — |
| **POSTPAY 카드 협의** | 카드 | 계약서 별도 명시 (고객 부담) | TBO |
| **대형 고객사 (TTV 5억+)** | 계약 협의 | 별도 청구 옵션 | TBO |
