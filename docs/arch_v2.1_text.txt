1. 비즈니스 문제 정의
1.1 닷비즈의 존재 이유
OhMyHotel은 현재 일 3,000건 이상의 부킹을 처리하고 있으나, 이는 전량 API 채널(예: Ctrip, Agoda 등 ~80개 채널)을 통해 발생한다. B2C 또는 B2B 직접 예약은 거의 0건이다.
이는 두 가지 구조적 문제를 의미한다:
채널 집중 리스크: Ctrip/Trip.com 단일 채널이 TTV의 44.3%를 차지. 해당 채널의 정책 변경이나 분쟁 시 OMH 매출의 절반이 위험에 처함.
마진 통제력 부재: API 채널을 통한 매출은 채널이 마진을 결정. OMH가 직접 에이전트에게 판매하면 마진 자율성 확보 가능.
닷비즈는 이 문제를 해결하는 신규 직접 예약 채널이다:

타겟 사용자
API 연동 없이 웹에서 직접 예약하는 중소형 여행사/에이전트
기존 API 부킹과의 관계
별개 채널. API 3,000건/일과 별도로 신규 직접 예약 매출 창출

2.1.2 검색 아키텍처: BFF Search Middleware
DOTBIZ BFF에 검색 전처리 미들웨어를 추가하여 ELLIS API로 전달하기 전에 검색어를 처리한다:
User Input          BFF Search Middleware          ELLIS Rate Engine
                                                                    
&apos;gracery shinjuku&apos;  → normalize(&apos;gracery shinjuku&apos;)              
                    → tokenize: [&apos;gracery&apos;, &apos;shinjuku&apos;]          
                    → Strategy 1: ELLIS API (tokens AND match)   
                       └→ 결과 있음 → 반환                        
                    → Strategy 2: Redis 호텔명 인덱스 (fuzzy)    
                       └→ trigram 유사도 &gt; 0.3 → 반환          
                    → Strategy 3: 도시 + 호텔명 분리 검색      
                       └→ fallback 결과 반환

2.1.4 호텔명 인덱스 (Redis)
BFF에서 ELLIS 호텔 데이터를 주기적으로 동기화하여 Redis에 검색용 인덱스를 구축한다:
데이터: 2,500+ 일본 직계약 호텔 + 아시아 7,000개 호텔명/ID/도시/에리어/등급
인덱스 키: hotel:{normalized_name}, city:{city_code}, area:{area_code}
동기화 주기: 일 1회 (ELLIS 호텔 DB → Redis)
검색 성능: Redis 인덱스 히트 시 &lt; 100ms, ELLIS API 펴백 시 &lt; 1초

3.2 셀프서비스 가입 플로우
신규 에이전트가 가입에서 첫 예약까지 최소한의 마찰로 도달하는 것이 핵심:
Step 1: Self-Registration (3-Step, v1.0 유지)
  └→ 회사정보 → 사용자정보 → 약관동의
  └→ Pending 상태로 계정 생성

Step 2: Internal Approval (OMH 내부)
  └→ 사업자등록 확인 + 신용 평가
  └→ 업체 유형 결정 (선불/후불) + Credit Line 설정
  └→ 목표: 가입 후 24시간 이내 승인 (SLA)

Step 3: First Booking (첫 예약 가이드)
  └→ 승인 완료 이메일에 &apos;첫 예약 가이드&apos; 포함
  └→ 로그인 시 온보딩 투어 (주요 기능 3분 안내)
  └→ AI Assistant에서 &apos;첫 예약 도우미&apos; Quick Action

6.1 인증 플로우
Client                    BFF                       ELLIS
  |-- POST /auth/login --&gt;|                           |
  |   (email + password)  |-- validate credentials --&gt;|
  |                       |&lt;-- user + role + tenant ---|
  |                       |-- JWT (15min) + Refresh --|
  |&lt;-- httpOnly Cookie ---|-- Redis session store ---&gt;|
  |                       |                           |
  |-- API call + Cookie -&gt;|-- verify JWT -----------&gt;|
  |                       |-- check RBAC (Master/OP) |
  |                       |-- check tenant isolation |
  |                       |-- call ELLIS API -------&gt;|
  |&lt;-- response ----------|&lt;-- data -----------------|
