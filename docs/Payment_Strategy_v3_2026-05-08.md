## ⚖️ 핵심 도전 과제

**계약 주체 = 싱가포르 / 결제 통화 다양 / 고객은 4개 권역에 분산**

→ 결제 흐름 설계 시 3가지 충돌:
1. **현지 결제 편의성** (가상계좌, Alipay 등) vs **싱가포르 단일 회계**
2. **수수료 최소화** (Local PG) vs **Cross-border 송금 비용**
3. **다통화** (KRW/CNY/VND/SGD/USD) vs **FX 단순화**

## 🎯 3가지 결제 수금 모델 비교

### Model A: 싱가포르 단일 입금 (Pure SG)

```
한국 OP → SWIFT 송금 → SG 본사 계좌 (USD)
중국 OP → SWIFT 송금 → SG 본사 계좌 (USD)
```

| 영역 | 효과 |
|------|------|
| 회계 | ✅ 매우 단순 (SG 단일) |
| FX | ⚠️ 모든 거래에 환산 |
| 비용 | ⚠️ SWIFT $25~50/건 + 0.1~0.5% FX |
| 고객 편의 | ❌ 현지 결제 수단 미사용 |
| 적합 | 대형 고객사 (월 TTV 5억+) |

### Model B: 현지 지사 수금 + 내부 송금 (Branch Collection)

```
한국 OP → 한국 지사 가상계좌 (KRW) → 한국 지사 → SG 본사 (월 1회 정산)
일본 OP → 일본 지사 가상계좌 (JPY) → 일본 지사 → SG 본사
베트남 OP → 베트남 지사 VNPay (VND) → 베트남 지사 → SG 본사
중국 OP → 홍콩 지사 Alipay/WeChat (HKD/CNY) → 홍콩 지사 → SG 본사
```

| 영역 | 효과 |
|------|------|
| 회계 | ⚠️ 내부 정산 처리 필요 |
| FX | ✅ 월 1회 일괄 환산 (헷지 가능) |
| 비용 | ✅ 가상계좌 0.025%, Alipay 1~2% |
| 고객 편의 | ✅ 현지 결제 수단 |
| 적합 | 일반 고객사 (소~중형) |

### Model C: 하이브리드 (권장) ⭐

대형/소형 고객사별 + 권역별 분기:

```
대형 고객사 (월 TTV 5억+)
  → SG 본사 직접 송금 (Model A)
  → 계약서 명시

일반 고객사
  ├ 한국·일본·베트남 OP → 현지 지사 수금 (Model B)
  ├ 중국·대만·홍콩 OP → 홍콩 지사 (오픈 후)
  └ SG·MY OP → SG 본사 직접

Cross-border 결제 (옵션)
  → Eximbay (한국 → 다통화 정산)
  → 2C2P (싱가포르 cross-border)
```

→ **Model C 채택 권장**

## 💳 권역별 결제 전략 (실측 데이터 기반)

### 권역 1: 한국

**고객 OP**: 한국 여행사 직원
**수금 주체**: 한국 지사 → 월 1회 SG 본사 송금

| 결제 수단 | 수수료 | 우선순위 | 비고 |
|----------|--------|---------|------|
| **가상계좌 (KG이니시스/KCP/토스)** | **건당 250원** | 🟢 **표준** | $1K 결제 시 0.025% |
| 신용카드 (Toss/KCP) | 2.1~3.3% | 🟡 Non-refundable 한정 | |
| 무통장 입금 | 0% | 🟢 일반 송금 | |
| Eximbay (대형) | 0.2~5% | 🟡 Cross-border 시 | 다통화 정산 가능 |
| SWIFT → SG | $25~50 + 0.1% | 🔴 대형사 한정 | 일반은 불리 |

**권장 흐름**:
```
OP → 한국 지사 가상계좌 (KRW)
    → 한국 지사 보관
    → 월말 일괄 USD 환산 → SG 본사 송금 (헷지 적용)
```

### 권역 2: 대중화권 (중국, 대만, 홍콩)

**고객 OP**: 중국/대만/홍콩 여행사
**수금 주체**: 홍콩 지사 (오픈 후) / SG 본사 (오픈 전)

| 결제 수단 | 수수료 | 우선순위 | 비고 |
|----------|--------|---------|------|
| **Alipay** (중국 발행) | ~0~1% | 🟢 **중국 표준** | B2B 협상 |
| **WeChat Pay** | ~0~1% | 🟢 중국 표준 | |
| **UnionPay** | 1.5~2.5% | 🟢 카드 | |
| FPS (홍콩 즉시 결제) | 0% | 🟢 홍콩 표준 | 홍콩 지사 오픈 후 |
| Alipay HK | 0~1% | 🟢 홍콩 | |
| 대만 카드 (CTBC 등) | 2~3% | 🟡 | |
| Alipay (해외 카드) | **3%** | 🔴 회피 | |

**권장 흐름**:
```
중국 OP → Alipay/WeChat → 홍콩 지사 (HKD/USD)
대만 OP → 카드/송금 → 홍콩 지사
홍콩 OP → FPS → 홍콩 지사
    → 홍콩 지사 → SG 본사 (월 1회)
```

⚠️ **홍콩 지사 오픈 전**: SG 본사 직접, Eximbay 또는 2C2P 활용

### 권역 3: 베트남 + 동남아 (태국, 인도네시아, 필리핀)

**고객 OP**: 동남아 여행사

| **인도네시아** | **Xendit** (3% + $0.30) | 3% | 🟢 SG 본사 |

## 📜 계약/회계 처리 가이드

### 인보이스 발행 주체
- **항상 SG 본사 (omh-sg)** — 계약 주체이므로
- 현지 지사는 "수금 위탁 (Collection Agent)" 역할
- 인보이스 보존: 7년 (SG 회계법 + 한국 전자세금계산서법 동시 충족)

### 내부 정산 (지사 → 본사)
- 월말 마감 → 익월 5일까지 SG 본사 송금
- FX는 월 평균 환율 (또는 송금일 환율)
- 지사 보유 잔액은 운영비/직원 급여로 일부 활용 가능

### Cross-border 결제 데이터
| 결제 흐름 | FX 처리 |
|---------|--------|
| 현지 가상계좌 → 지사 → SG | 송금 시점 환율 |
| 현지 PG → SG 직접 | PG 정산일 환율 |
| SWIFT → SG | 송금 시점 환율 |

## 🎯 마진 시뮬레이션 (Model C 적용)

### 가정
- Gross Margin: 4%
- ELS: 평균 0.6% (Tier 가중 평균)
- CMS: 1.0%
- 평균 결제 수수료 (Model C): **1.0%** (가상계좌+Local PG 가중)

```
Gross Margin:        4.00%
− ELS:              -0.60%
− CMS:              -1.00%
− 결제 수수료:        -1.00%
─────────────────────────────
Net Margin:         +1.40% ✅
```

→ **마진 4% 환경에서도 Model C 적용 시 Net +1.4% 확보**

### 권역별 Net Margin (실측)

| 권역 | 평균 결제 수수료 | Net Margin |
|------|---------------|------------|
| 한국 (가상계좌) | 0.025% | **+2.4%** |
| 대중화권 (Alipay 등) | 1.0% | +1.4% |
| 베트남+동남아 (QR) | 1.5% | +0.9% 🟡 |
| SG+MY (PayNow) | 0% | **+2.4%** |

→ 동남아만 다소 압박, 나머지는 안전.
